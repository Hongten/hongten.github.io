LeetCode 1020: Number of Enclaves (Border Flood Fill)

Today we solve LeetCode 1020 - Number of Enclaves.

Source: https://leetcode.com/problems/number-of-enclaves/

English

Problem Summary

Given a binary grid, 1 means land and 0 means sea. You can move up/down/left/right from land. Return how many land cells cannot walk off the boundary of the grid.

Key Insight

Instead of directly finding enclosed land, remove all land reachable from the border first. The remaining land cells are exactly enclaves.

Algorithm

- Start DFS/BFS from every border land cell and mark visited land as sea.
- After flood fill, scan the whole grid.
- Count cells still equal to 1.

Complexity Analysis

Let m be rows and n be columns.
Time: O(mn).
Space: O(mn) worst case due to recursion stack/queue.

Common Pitfalls

- Forgetting to process all four borders.
- Revisiting cells without marking early.
- Counting before border-connected land is fully removed.

Reference Implementations (Java / Go / C++ / Python / JavaScript)

class Solution {
    private int m, n;
    private int[][] grid;
    private final int[] dr = {1, -1, 0, 0};
    private final int[] dc = {0, 0, 1, -1};

    public int numEnclaves(int[][] grid) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;

        for (int r = 0; r < m; r++) {
            dfs(r, 0);
            dfs(r, n - 1);
        }
        for (int c = 0; c < n; c++) {
            dfs(0, c);
            dfs(m - 1, c);
        }

        int ans = 0;
        for (int r = 0; r < m; r++) {
            for (int c = 0; c < n; c++) {
                if (grid[r][c] == 1) ans++;
            }
        }
        return ans;
    }

    private void dfs(int r, int c) {
        if (r < 0 || r >= m || c < 0 || c >= n || grid[r][c] == 0) return;
        grid[r][c] = 0;
        for (int k = 0; k < 4; k++) {
            dfs(r + dr[k], c + dc[k]);
        }
    }
}

    def numEnclaves(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])

        def dfs(r: int, c: int) -> None:
            if r < 0 or r >= m or c < 0 or c >= n or grid[r][c] == 0:
                return
            grid[r][c] = 0
            dfs(r + 1, c)
            dfs(r - 1, c)
            dfs(r, c + 1)
            dfs(r, c - 1)

        for r in range(m):
            dfs(r, 0)
            dfs(r, n - 1)
        for c in range(n):
            dfs(0, c)
            dfs(m - 1, c)

        return sum(grid[r][c] == 1 for r in range(m) for c in range(n))
var numEnclaves = function(grid) {
  const m = grid.length, n = grid[0].length;
  const dirs = [[1, 0], [-1, 0], [0, 1], [0, -1]];

  const dfs = (r, c) => {
    if (r < 0 || r >= m || c < 0 || c >= n || grid[r][c] === 0) return;
    grid[r][c] = 0;
    for (const [dr, dc] of dirs) {
      dfs(r + dr, c + dc);
    }
  };

  for (let r = 0; r < m; r++) {
    dfs(r, 0);
    dfs(r, n - 1);
  }
  for (let c = 0; c < n; c++) {
    dfs(0, c);
    dfs(m - 1, c);
  }

  let ans = 0;
  for (let r = 0; r < m; r++) {
    for (let c = 0; c < n; c++) {
      if (grid[r][c] === 1) ans++;
    }
  }
  return ans;
};

中文

题目概述

给定一个二进制网格，1 代表陆地，0 代表海洋。你可以在陆地上四联通移动。返回无法走到网格边界之外的陆地单元数量。

核心思路

与其直接找“飞地”，不如先从边界出发淹没所有可达陆地。最后剩下的陆地就是飞地。

算法步骤

- 对所有边界上的陆地做 DFS/BFS，并标记为海洋。
- 完成后遍历全图。
- 统计仍为 1 的格子数。

复杂度分析

设网格大小为 m x n。
时间复杂度：O(mn)。
空间复杂度：O(mn)（递归栈或队列最坏情况）。

常见陷阱

- 漏掉某一条边界的起点。
- 没有及时标记导致重复访问。
- 在边界清理完成前就开始计数。
