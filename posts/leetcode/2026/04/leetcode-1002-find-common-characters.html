LeetCode 1002: Find Common Characters (Per-Letter Minimum Frequency Intersection)

2026-04-06 · LeetCode · String / Counting

Today we solve LeetCode 1002 - Find Common Characters.

Source: https://leetcode.com/problems/find-common-characters/

English

Problem Summary

Given an array of lowercase strings words, return all characters that appear in every string, including duplicates.

Key Insight

For each character a..z, its final multiplicity is the minimum frequency across all words. So we keep a global count array and intersect it with each word using min.

Brute Force and Limitations

A brute-force idea is repeatedly removing matched chars among strings, but string deletion is inefficient and messy. Counting arrays are simpler and faster.

Optimal Algorithm Steps (26-Count Intersection)

1) Initialize global[26] with frequencies from words[0].
2) For every next word, build local[26].
3) Update global[i] = min(global[i], local[i]) for all 26 letters.
4) Build answer by appending each letter global[i] times.

Complexity Analysis

Time: O(total_chars + 26 * n), effectively linear in input size.
Space: O(26) extra (excluding output).

Common Pitfalls

- Using set intersection (loses duplicates).
- Forgetting to reset local counts for each word.
- Returning chars in arbitrary order; expected output is usually alphabetic by construction.

Reference Implementations (Java / Go / C++ / Python / JavaScript)

class Solution {
    public List<String> commonChars(String[] words) {
        int[] global = new int[26];
        Arrays.fill(global, Integer.MAX_VALUE);

        for (String w : words) {
            int[] local = new int[26];
            for (char c : w.toCharArray()) {
                local[c - 'a']++;
            }
            for (int i = 0; i < 26; i++) {
                global[i] = Math.min(global[i], local[i]);
            }
        }

        List<String> ans = new ArrayList<>();
        for (int i = 0; i < 26; i++) {
            for (int k = 0; k < global[i]; k++) {
                ans.add(String.valueOf((char) ('a' + i)));
            }
        }
        return ans;
    }
}
func commonChars(words []string) []string {
    global := make([]int, 26)
    for i := 0; i < 26; i++ {
        global[i] = 1 << 30
    }

    for _, w := range words {
        local := make([]int, 26)
        for i := 0; i < len(w); i++ {
            local[w[i]-'a']++
        }
        for i := 0; i < 26; i++ {
            if local[i] < global[i] {
                global[i] = local[i]
            }
        }
    }

    ans := make([]string, 0)
    for i := 0; i < 26; i++ {
        for k := 0; k < global[i]; k++ {
            ans = append(ans, string(rune('a'+i)))
        }
    }
    return ans
}

class Solution:
    def commonChars(self, words: List[str]) -> List[str]:
        global_cnt = [float('inf')] * 26

        for w in words:
            local = [0] * 26
            for ch in w:
                local[ord(ch) - ord('a')] += 1
            for i in range(26):
                global_cnt[i] = min(global_cnt[i], local[i])

        ans = []
        for i in range(26):
            ans.extend([chr(ord('a') + i)] * int(global_cnt[i]))
        return ans
/**
 * @param {string[]} words
 * @return {string[]}
 */
var commonChars = function(words) {
  const global = new Array(26).fill(Number.MAX_SAFE_INTEGER);

  for (const w of words) {
    const local = new Array(26).fill(0);
    for (const ch of w) {
      local[ch.charCodeAt(0) - 97]++;
    }
    for (let i = 0; i < 26; i++) {
      global[i] = Math.min(global[i], local[i]);
    }
  }

  const ans = [];
  for (let i = 0; i < 26; i++) {
    for (let k = 0; k < global[i]; k++) {
      ans.push(String.fromCharCode(97 + i));
    }
  }
  return ans;
};

中文

题目概述

给定一个只含小写字母的字符串数组 words，返回所有在每个字符串中都出现的字符（需要保留重复次数）。

核心思路

对于每个字母 a..z，它在答案中的出现次数就是它在所有单词中的最小出现次数。因此用 26 长度计数数组做“按位取最小值”的交集即可。

暴力解法与不足

暴力做法是不断在字符串里查找并删除匹配字符，逻辑复杂且效率差。计数法更直接、更稳定。

最优算法步骤（26 维频次数组求交）

1）用 words[0] 初始化全局频次 global[26]。
2）遍历后续每个单词，统计其局部频次 local[26]。
3）对每个字母执行 global[i] = min(global[i], local[i])。
4）最后把每个字母按 global[i] 次加入答案。

复杂度分析

时间复杂度：O(total_chars + 26 * n)，本质上是线性。
空间复杂度：O(26)（不含输出）。

常见陷阱

- 只做集合交集会丢失重复字符。
- 忘记每轮重置 local 计数数组。
- 输出顺序混乱；按字母顺序构造答案最稳妥。
