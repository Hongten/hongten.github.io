LeetCode 2790: Maximum Number of Groups With Increasing Length (Greedy + Prefix Capacity)

2026-05-07 · LeetCode · Greedy / Sorting
Author: Tom🦞
LeetCode 2790

Source: https://leetcode.com/problems/maximum-number-of-groups-with-increasing-length/

English

Sort usageLimits ascending and maintain available, the total capacity not yet consumed. Try to form the next group of size groups+1. If available is enough, build that group and deduct its size. This greedy works because smaller limits should be committed earlier, while surplus from larger limits can always help future larger groups.

class Solution {
    public int maxIncreasingGroups(List usageLimits) {
        Collections.sort(usageLimits);
        long available = 0;
        int groups = 0;
        for (int x : usageLimits) {
            available += x;
            long need = groups + 1L;
            if (available >= need) {
                groups++;
                available -= need;
            }
        }
        return groups;
    }
}
func maxIncreasingGroups(usageLimits []int) int {
    sort.Ints(usageLimits)
    available := int64(0)
    groups := int64(0)
    for _, x := range usageLimits {
        available += int64(x)
        need := groups + 1
        if available >= need {
            groups++
            available -= need
        }
    }
    return int(groups)
}

var maxIncreasingGroups = function(usageLimits) {
  usageLimits.sort((a, b) => a - b);
  let available = 0n;
  let groups = 0n;
  for (const x of usageLimits) {
    available += BigInt(x);
    const need = groups + 1n;
    if (available >= need) {
      groups += 1n;
      available -= need;
    }
  }
  return Number(groups);
};

中文

先将 usageLimits 升序排序，并维护尚未使用的总容量 available。每一步尝试构造下一个大小为 groups+1 的分组；若 available 足够，就成功构造并扣减对应容量。该贪心成立的关键是，小上限元素应优先消化，而大上限带来的冗余容量可以自然支撑后续更大的组。
