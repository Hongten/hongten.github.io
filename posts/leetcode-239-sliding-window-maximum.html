Today we solve LeetCode 239 - Sliding Window Maximum.

Source: https://leetcode.com/problems/sliding-window-maximum/

English

Problem Summary

Given an integer array nums and window size k, return an array containing the maximum value in each sliding window as it moves from left to right by one step.

Key Insight

Maintain a monotonic decreasing deque of indices. The front always stores the index of the current window maximum. Before inserting a new index, pop smaller values from the back because they can never become maximum again.

Brute Force and Limitations

Brute force checks every window and scans all k elements to find max, costing O(nk). For large arrays this is too slow.

Optimal Algorithm Steps

1) Use deque to store indices, and keep values decreasing by index order.
2) For each index i, remove front if it is outside current window (<= i-k).
3) Remove back while nums[back] <= nums[i].
4) Push i to back.
5) When i >= k-1, record nums[deque.front] as window max.

Complexity Analysis

Time: O(n). Each index enters and exits deque at most once.
Space: O(k) for deque.

Common Pitfalls

- Storing values instead of indices, making expiry checks difficult.
- Forgetting to remove expired index before reading max.
- Using strict < instead of <= when popping back can keep redundant equal values (still correct, but less clean).

class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        if (n == 0 || k == 0) return new int[0];

        int[] ans = new int[n - k + 1];
        Deque<Integer> dq = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            if (!dq.isEmpty() && dq.peekFirst() <= i - k) dq.pollFirst();

            while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) dq.pollLast();
            dq.offerLast(i);

            if (i >= k - 1) ans[i - k + 1] = nums[dq.peekFirst()];
        }
        return ans;
    }
}
func maxSlidingWindow(nums []int, k int) []int {
    n := len(nums)
    if n == 0 || k == 0 {
        return []int{}
    }

    deque := make([]int, 0)
    ans := make([]int, 0, n-k+1)

    for i := 0; i < n; i++ {
        if len(deque) > 0 && deque[0] <= i-k {
            deque = deque[1:]
        }

        for len(deque) > 0 && nums[deque[len(deque)-1]] <= nums[i] {
            deque = deque[:len(deque)-1]
        }

        deque = append(deque, i)

        if i >= k-1 {
            ans = append(ans, nums[deque[0]])
        }
    }

    return ans
}
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> dq;
        vector<int> ans;

        for (int i = 0; i < (int)nums.size(); ++i) {
            if (!dq.empty() && dq.front() <= i - k) dq.pop_front();

            while (!dq.empty() && nums[dq.back()] <= nums[i]) dq.pop_back();
            dq.push_back(i);

            if (i >= k - 1) ans.push_back(nums[dq.front()]);
        }
        return ans;
    }

  const deque = []; // store indices
  const ans = [];

  for (let i = 0; i < nums.length; i++) {
    if (deque.length && deque[0] <= i - k) deque.shift();

    while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) {
      deque.pop();
    }

    deque.push(i);

    if (i >= k - 1) ans.push(nums[deque[0]]);

中文

题目概述

给定整数数组 nums 和窗口大小 k，窗口每次向右移动一格，返回每个窗口中的最大值。

核心思路

用单调递减双端队列（存下标）维护窗口候选最大值：队首永远是当前窗口最大值下标。新元素进入前，把队尾比它小（或相等）的元素弹出。

暴力解法与不足

暴力法对每个窗口扫描 k 个元素取最大值，复杂度 O(nk)，数据大时性能不足。

最优算法步骤

1）队列里只存下标，并保持对应值单调递减。
2）每到新位置，先弹出过期下标（不在窗口内）。
3）再弹出所有比当前值小/等于的队尾下标。
4）将当前下标入队。
5）当窗口形成（i >= k-1）时，队首即当前窗口最大值。

复杂度分析

时间复杂度：O(n)，每个元素最多进出队列一次。
空间复杂度：O(k)。

常见陷阱

- 队列里存值不存下标，导致无法判断是否过期。
- 忘记先处理过期元素就直接读队首。
- 边界处理错误：k=1、数组长度与窗口关系等。
